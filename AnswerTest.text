
ตอบคำถาม 1.3 Technology ที่เลือกใช้ พร้อมคำอธิบายในการเลือกใช้

ส่วนของหน้าบ้านมี 2 ตัวเลือกที่ดี
1. ใช้ React ในการพัฒนา UI รวดเร็วตอบโจทย์ยิ่งเป็น ระบบการขายตั๋วเครื่องบินของสายการบิน ฉะนั้นจะมี action ของ user
ที่เยอะ อีกทั้งในอนาคตหากอยากทำ app มือถือก็สามารถใช้ React-Native ได้เลย อีกอย่างหากมีปัญหาเรื่อง technical React เป็น framework
ที่คนใช้เยอะ search หาก็เยอะได้ไม่ยาก ตอบโจทย์ทั้ง user และ dev ครับ

2. ใช้ Augular ในการพัฒนา UI รวดเร็วเช่นกัน แต่อาจจะมีผลเสียเรื่อง ความจำในการติดตั้งที่ค่อยข้างเยอะกว่า React เล็กน้อยครับ
เป็น framework ที่คนใช้เยอะเหมือนกันครับ search หาก็เยอะได้ไม่ยาก ตอบโจทย์ทั้ง user และ dev ครับ

ส่วนของหลังบ้านใช้ Node.js และ Express ก็ได้ครับ ที่ใช้ 2 ตัวนี้เพราะ เป็นพื้นฐานภาษามาจาก Javascript เข้าใจได้ง่าย คนใช้เยอะ
Database ที่ใช้จะเป็น Relational Database SQL เพราะข้อมูลจะได้เป็นระเบียบมีความเชื่อมโยงกันระหว่างข้อมูลครับ ทำให้เวลาจะดึงข้อมูลมาใช้ จะสามารถทำได้ง่าย (postgresql หรือ oracel ก็ได้ครับ ต่างกันแค่ syntax ในการ dev)

ตอบคำถาม 1.4 ในกรณีที่มีการซื้อตั๋วในช่วงปีใหม่จำนวนมากๆ ให้อธิบายปัญหาที่อาจจะเกิดขึ้นพร้อมวิธีแก้ปัญหา

ปัญหาที่อาจจะพบ:
1.ระบบอาจล่ม : การเข้าถึงระบบพร้อมกันจำนวนมากอาจทำให้เซิร์ฟเวอร์ไม่สามารถรองรับได้ และเกิดการล่มหรือทำงานช้า
2.ปัญหาการจองซ้ำซ้อน : การทำธุรกรรมหลายรายการพร้อมกันอาจทำให้เกิดการจองซ้ำซ้อนหรือความผิดพลาดในการจัดการที่นั่ง
3.ปัญหาการจ่ายเงิน : ระบบอาจไม่สามารถประมวลผลการชำระเงินได้ทันทีเนื่องจากความต้องการสูง ทำให้ลูกค้าอาจต้องพยายามชำระเงินซ้ำหลายครั้ง

วิธีการแก้ไข:
1.เพิ่มขีดความสามารถของเซิร์ฟเวอร์ : การเพิ่มจำนวนเซิร์ฟเวอร์หรือการใช้ระบบคลาวด์ เพื่อเพิ่มขีดความสามารถในการรองรับผู้ใช้งานพร้อมกัน
2.การกระจายโหลด : ใช้เทคนิคการกระจายโหลดเพื่อลดความหนาแน่นของการเข้าถึงในเซิร์ฟเวอร์เดียว
3.การเก็บคิว : ใช้ระบบคิวเพื่อจัดการกับผู้ใช้งานที่รอการจองตั๋ว ป้องกันการเข้าถึงพร้อมกันเป็นจำนวนมาก
4.การสำรองระบบ : จัดให้มีระบบสำรองกรณีที่ระบบหลักเกิดปัญหา
5.การจำลองสถานการณ์ : ทำการทดสอบความสามารถของระบบด้วยการจำลองการเข้าถึงจำนวนมากก่อนถึงช่วงเทศกาล เพื่อระบุจุดอ่อนของระบบและปรับปรุงก่อนใช้งานจริง
